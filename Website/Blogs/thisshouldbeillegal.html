The Thing

So, while browsing the web just a few months ago (for Internet TV links), I found this gem on GitHub, of all places:


Yee.

Anyway, in short, it's basically a hot tub of neatly organized and suspicious IPTV links. Like links that start with .vip (looks familliar?). It was a fun thing, because it gave you basically the entire world's TV stations right at your fingertips. And because of this, it got in trouble a couple times.

The troubling starts...

Back in Feburary, the page actually got shut down completely because it was using too much resources. And get this: the email that got sent to the uploaders said they were violating the DMCA (or the quote-unquote, "Digital Millenium Copyright Act"). And they were actually pretty confused, 'cause they never did knowingly uploaded any copyrighted links.

Anyway, GitHub eventually clarified that it was actually "abusing" GitHub Actions (that was used to dynamically update the links, apparently), so the maintainers eventually did just do the simple way: disabing GitHub Actions.

Anyway, that interesting button...


Huh, why is it there now?

Yep, that website above is their interface for accessing the Internet TV Links. It used to only have a search bar and a couple buttons, but now it has added another very interesting button... Create Playlist.

So I did what any rational person would do: click it and see what happens next!

What does it do?

Apparently, that button links you to an interface, in which you can select tv channels and add them to a playlist. I would just have a logical guess that this idea stems from that discussion post in Febuary, because someone has the brilliant idea to make your own playlists and Electronic Program Guides' xml files.


Tee hee hee!

Anyway, after selecting the TV channels and clicking Download Playlist, it downloads a file called playlist.m3u.


Hey look, a new playlist just dropped!

This is what's basically an HTTP Live Stream file (or, to put it another way: it tells your media player what video file it's going to play and the metadata (if your media player supports it, that is. The only players that support it is a TV streaming server.) So yeah, it's basically that. )


Just like a HTML document! (in a different format, that is)

Let's feed this into a media player and see what happens!

Even though I am definitely not using RealPlayer 5.0 for this one, I'll be using *drum roll please...* Windows 11's default Media Player to try this out!

The only absolute reason I could give to do this is that it supports HLS. That's it. Or I would've used ffplay instead.


Oh, there it goes!

So yeah, it works, just that... well, it might be illegal.
